# Domain Docs

InsightForge is a single-context repository.

## Before exploring

- Read the root domain glossary.
- Read the system-wide ADRs relevant to the area being changed.
- Read the testing strategy before adding or modifying tests.

Proceed silently if a referenced category of document does not yet exist.

## Vocabulary

Use canonical terms from the domain glossary in code, tests, issues, and documentation. Do not drift to synonyms that the glossary explicitly marks to avoid.

If required language is missing, either reconsider whether the concept belongs to the domain or use the domain-modeling workflow to resolve and record it.

## Decisions

Surface conflicts with existing ADRs explicitly rather than silently overriding them. Add a new ADR only for a hard-to-reverse, surprising decision that resolves a genuine trade-off.
