# Keep generated artifacts read-only

InsightForge treats generated artifacts as read-only and makes repeated Stage Runs a primary interaction. When a result needs a Project-specific correction, the Author improves its upstream source; when it reveals a reusable weakness, the Author improves the shared Stage Prompt. This is deliberately more demanding than direct editing because the product exists to help Authors improve a repeatable product-development workflow, not merely polish one generated document.

## Consequences

The interface must make iteration fast, preserve prior results, and clearly guide the Author toward either improving the source or improving the prompt before rerunning a stage.
